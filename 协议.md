您理解得完全正确！要开发工控屏（HMI）程序并与这块基于 Linux 的主控板完美对接，核心就是**吃透双方的“串口通信协议”**。

根据您提供的 `ui_get.c`、`ui_get.h` 和 `enum_uartparam_addr.h` 源码，主控板与工控屏之间采用的是一套**自定义的、基于一字节对齐（`#pragma pack(1)`）、大端模式（Big-Endian, 使用 `htons/htonl`）的二进制通信协议**。

我为您梳理出了实现工控屏系统所必需的**三大协议核心模块**，您可以直接将此作为工控屏端串口解析状态机的开发文档：

---

### 一、 协议基础帧格式 (全局通用)

不管是主控板发给屏幕，还是屏幕发给主控板，所有数据帧都遵循以下物理结构：

* **帧头 (2 Bytes)**：`0xD5 0x5D` (程序中通过 `htons(0xD55D)` 转换)。
* **命令字/功能码 (1 Byte)**：如 `0x04`、`0x05` 等。
* **消息类型 (1 Byte)**：`0xFF` (主动上报)，`0x00` (应答成功)，`0x01` (应答失败)。
* **数据域 (N Bytes)**：变长载荷（各命令的数据结构不同）。
* **校验和 CRC16 (2 Bytes)**：从“命令字”开始，到“数据域”结束的 CRC-16 校验码。
* **帧尾 (2 Bytes)**：`0x5D 0xD5` (程序中通过 `htons(0x5DD5)` 转换)。

> **💡 工控屏开发提示 (CRC算法)**：工控屏端计算 CRC 时，校验范围**不包含帧头、CRC本身和帧尾**（对应源码中的 `BYTE_SIZE - 6`），且多项式为 `0xA001`。

### 二、 主控板 -> 工控屏 (屏幕需“接收并显示”的周期指令)

在主控板的 `write_ui_Thread` 线程中，会**每秒周期性**向工控屏发送以下指令。工控屏需要根据“命令字”解析并在 UI 上渲染：

1. **`0x08` 开机显示参数 (基础设置)**
* **作用**：告诉屏幕当前电台的基础配置。
* **解析内容**：当前时间、工作模式（定频/自适应）、跳频中心频点（频率1-4）、带宽（bw）、调制方式（mcs）、内外同步模式等。


2. **`0x05` 设备基础信息 (定位与 IP)**
* **作用**：电台向屏幕汇报自己的身份和 GPS 定位。
* **解析内容**：设备 ID（`kddt-X`）、业务网口 IP、高精度 GPS 经度、纬度、高度、以及当前是否锁定卫星时间。


3. **`0x06` 流量收发统计**
* **作用**：用于在屏幕上显示网口和话音的实时吞吐量。
* **解析内容**：以太网发送/接收包数 (`eth_tx_cnt`/`eth_rx_cnt`)、模拟话音发送/接收包数 (`voice_tx_cnt`/`voice_rx_cnt`)。


4. **`0x07` 系统自检状态 (硬件监控)**
* **作用**：硬件健康状态上报，用于屏幕的告警模块。
* **解析内容**：电池电量 (`battery_remaining_capacity`)、主板温度、风扇状态、卫导锁定状态、各射频通道的发射功率检波和驻波状态（VSWR）等。


5. **`0x09` 在网节点列表 (雷达拓扑图数据)**
* **作用**：核心数据！用于在屏幕上画出周围的邻居节点连线图。
* **解析内容**：这是一个**变长报文**。它会包含 `neigh_num` 个邻居信息，每个包含：邻居 ID、IP地址、信号强度 (`signal_strength`，即 -RSSI)、传输延迟。

### 三、 工控屏 -> 主控板 (屏幕需“主动发送”的控制指令)

当用户在工控屏上点击“修改频率”、“修改带宽”等按钮时，工控屏需要向主控板发送特定的**寄存器地址修改指令**。

主控板的 `process_uart_info` 负责解析这段指令。工控屏下发的数据载荷格式非常严格：

* **长度字节 `length` (1 Byte)**：它的值为 `1(命令字) + 4(地址长度) + 真实Value的长度`。
* **寄存器地址 `value_addr` (4 Bytes)**：使用大端模式 (`htonl`)。由 `enum_uartparam_addr.h` 定义。
* **参数值 `value` (1, 2 或 4 Bytes)**：实际要修改的值。

#### 工控屏需要掌握的核心下发地址集：

*(屏幕下发以下地址和值，主控板会自动回复 `0x02` 的 ACK 帧)*

* **修改工作模式**：
* 发送地址：`0x12340000` (`PARAM_CH1_FREQ_HOPPING_MODE`)
* 值：`0` (定频) 或 `1` (自适应选频)


* **修改中心频点 (定频模式)**：
* 发送地址：`0x12351000` (`PARAM_CH1_FIXED_FREQ_CENTER`)
* 值：例如要设置 `2500` 兆赫兹，发送 `2500000` 的 4 字节大端格式


* **修改自适应选频列表**：
* 地址 `0x12352000` 到 `0x12355000` 分别对应 4 个备选频点。


* **修改路由协议**：
* 发送地址：`0x12310000` (`PARAM_CH1_ROUTING_PROTOCOL`)
* 值：`0` (OLSR), `1` (AODV), `2` (BATMAN)


* **修改信号带宽 / 调制方式(MCS)**：
* 带宽地址：`0x12370000` (`PARAM_CH1_SIGNAL_BANDWIDTH`)
* MCS 地址：`0x12382000` (`PARAM_CH1_MODULATION_WIDEBAND`)


* **清零流量统计**：
* 发送地址：`0x23900000` (`PARAM_TXRX_INFO_OPERATION`)
* 值：`2` (代表执行清零操作)

### 🚀 给工控屏开发者的架构建议：

在开发工控屏（如使用 Qt、C# 或基于串口屏如迪文/淘晶驰的脚本）时，您的串口接收循环必须写成**有限状态机 (FSM)**：

1. **寻头**：死等 `0xD5`，接着必须是 `0x5D`。
2. **分类**：读出第 3 个字节（命令字 `0x04~0x0A`），以此决定采用哪个结构体来反序列化。
3. **校验**：读满结构体长度后，屏蔽最后 4 个字节，计算 CRC。若与报文自带的 CRC 匹配，且最后两字节为 `0x5D 0xD5`，则说明收到一帧完美数据，更新屏幕 UI。
