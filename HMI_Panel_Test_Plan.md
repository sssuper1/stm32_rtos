# 通用多级菜单工控屏 (STM32+RTOS) 测试与验收文档

## 1. 测试环境与硬件连接
* **主控板**：STM32F103 (或通过串口模块连接开发板)

**管脚分配表 (Pinout) :**
* **OLED 显示屏 (I2C2 接口)**:
  * SCL: `PB10`
  * SDA: `PB11`
  * VCC: 3.3V
  * GND: GND
* **4x4 矩阵键盘**:
  * 行引脚 (输出 / Rows): 
    * R1: `PB12`
    * R2: `PB13`
    * R3: `PB14`
    * R4: `PB5`
  * 列引脚 (输入读取 / Cols):
    * C1: `PB6`
    * C2: `PB7`
    * C3: `PB8`
    * C4: `PB9`
* **USART1 主通信接口 (接 Linux 主板或 USB-TTL)**
  * TX: `PA9`
  * RX: `PA10`
  * GND: 必须与主板/TTL共地
  * 波特率：`115200`，`8N1`
* **心跳指示灯 (可选)**:
  * LED_HEARTBEAT: `PC13`

## 2. 第1阶段：开机与 UI 结构体基础测试 (本地脱机测试)
**目的**：验证 FreeRTOS 任务调度、OLED 驱动以及菜单树结构逻辑。

* **[Case 1.1] 开机启动测试**
  * 步骤：给板子通电。
  * 预期现象：几秒钟内 OLED 屏幕应稳定点亮，并显示根菜单主级 `MAIN`，第一行应展示 `OPERATION` 闪烁或被大框选中。

* **[Case 1.2] 按键消抖与基础导航逻辑**
  * 步骤：
    1. 按下**“向下”**键，预期光标选中 `FUNCTION`；再按下降选中 `SETTING`。
    2. 停留在 `FUNCTION` 并按下**“确认”**键，预期进入子级菜单，看到 `MEMBERS` 等字样。
    3. 按下**“返回”**键，预期退回到 `MAIN` 根菜单。
  * 检测标准：按键无二次连点（防抖可靠），页面切换必须是瞬发无残影、无闪烁。

* **[Case 1.3] 本地菜单焦点切换与编辑态 (Edit Mode) 防冲突测试**
  * 步骤：
    1. 导航进入 `OPERATION` 菜单，屏幕首先显示 `OP MODE` 及其当前参数值。
    2. 此时按下**“向下”**键，预期不应修改参数值，而是将菜单焦点从 `OP MODE` 平级切换到 `CHANNEL SET`（或其子项 `CH FREQ`），展示相关数据。
    3. 返回到 `OP MODE`，按下 **`*` (KEY_STAR)** 键进入修改/编辑模式。
    4. 此时按上下键执行增加或减小工作模式编号操作。
    5. 修改完毕后，按下“确认(OK)”或“返回(BACK)”键退出编辑模式。
  * 预期现象：未按 `*` 键时，上下导航不被底层数值修改操作阻断；按 `*` 键进入编辑态后，编辑数值应在限定区间内循环，且编辑反应顺滑。

## 3. 第2阶段：下行被动接收与互斥锁压力测试 (串机测试)
**目的**：验证主动上报流下，工控屏防死锁与刷新正确率。

* **[Case 2.1] 串口解析与字典自动更新**
  * 步骤：
    1. 导航进入 `FUNCTION` -> `TX/RX STATS` 页面（此时设备未收到数据，数值通常为 0）。
    2. 打开 PC 端串口助手或使用 Python 测试脚本，每隔 `1秒` 向设备连续发送包含 `0xD5 0x5D ...` 校验头的合法报文（例如模拟包含网口 TX/RX 数据包计数的业务帧 `0x08` 或状态包）。
  * 预期现象：OLED 上的数据不需要手动按键刷新，每秒自动“跳变”至最新串口发送的计数数值。

* **[Case 2.2] RTOS 抢占与死锁压力测试**
  * 步骤：
    1. 通过串口助手**极高频率（比如 50Hz, 10ms）** 连续向设备狂刷状态报文。
    2. 测试人员在此极高频干扰下，强行在键盘上输入、狂按快速翻页并在 `MEMBERS` 等带参数读取的页面逗留。
  * 预期现象：由于加入了高优先级 DMA 与 `gParamMutexHandle` 互斥保护，设备绝不能发生活锁(死机)；OLED 界面也不能因为读取残缺数据发生 "乱码/撕裂"，且 UI 按键操作应该能勉强保持响应。

## 4. 第3阶段：上行主动回传与协议组包测试 (闭环联调)
**目的**：验证对用户触发的参数修改操作，设备是否能成功发起 `ParamUpdateQueue` 到 UART 的正确转换。

* **[Case 3.1] 参数下发触发与队列唤醒**
  * 步骤：
    1. 将 STM32 TX 接到串口助手的 RX，准备抓包。
    2. 在面板上进入 `OPERATION` -> `OP MODE`，并进行数值修改。修改完成后，按下**“确认”**键。
  * 预期现象：修改指令（含对应键值的 `PARAM_ID_WORK_MODE` 及修改数据），将被投递至 FreeRTOS 队列。
  * 检查：串口助手会立刻捕获到一帧开头的以 `0xD5 0x5D ...` 打头，内部含有 `0x01`（参数下发 cmd）的二进制流报文。人工或脚本校验 CRC 及 Payload 数据结构是否符合通信手册设计。
